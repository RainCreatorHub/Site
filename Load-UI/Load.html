Documentação Fumadocs Style

Bem-vindo à documentação do projeto, construída com MDX, TypeScript e CSS moderno. Esta documentação demonstra como criar uma experiência similar ao Fumadocs.

Recursos Incluídos

  • Suporte a MDX para conteúdo dinâmico
  • Componentes TypeScript demonstrativos
  • Design responsivo com modo claro/escuro
  • Sintaxe highlight para código
  • Navegação lateral e sistema de busca

Como usar esta documentação

Esta documentação é um único arquivo HTML que você pode implantar diretamente no GitHub Pages. Ela inclui tudo que você precisa para começar.

Instalação

Para usar este template de documentação, siga estes passos:

npm
Yarn
Manual
npm install minha-biblioteca
# ou
npx create-documentation@latest
yarn add minha-biblioteca
# ou
yarn create documentation
git clone https://github.com/exemplo/documentacao.git
cd documentacao
cp -r template/ meu-projeto/
cd meu-projeto

Nota: Este template não requer instalação de dependências, pois tudo está incluído em um único arquivo HTML.

Exemplo de Interface TypeScript 
interface User {
  id: number;
  name: string;
  email: string;
  role: 'admin' | 'user' | 'guest';
  
  // Método opcional
  getProfile?(): UserProfile;
}

interface UserProfile {
  avatar: string;
  bio: string;
  website?: string; // Opcional
}

// Função que utiliza a interface
function createUser(user: Partial<User>): User {
  return {
    id: Date.now(),
    name: 'Novo Usuário',
    email: '',
    role: 'user',
    ...user
  };
}

Usando MDX para Conteúdo Dinâmico

MDX permite que você escreva JSX em seu conteúdo Markdown. Aqui está um exemplo:

import { Chart } from '../components/Chart.jsx';

# Dashboard de Análise

Abaixo está um gráfico gerado dinamicamente:

<Chart 
  title="Visitas por Mês" 
  data={[120, 190, 300, 500, 200, 300]} 
/>

## Como personalizar

Você pode passar diferentes propriedades para o componente:

<Chart
  title="Taxa de Conversão"
  type="line"
  data={[0.1, 0.15, 0.2, 0.18, 0.22, 0.25]}
  color="#2563eb"
/>

Info: MDX combina a simplicidade do Markdown com o poder do JSX, permitindo componentes interativos em sua documentação.

Componentes Personalizados

// components/Callout.jsx
export function Callout({ type = 'note', children }) {
  const typeStyles = {
    note: { borderColor: '#eab308', bgColor: 'rgba(234, 179, 8, 0.1)' },
    warning: { borderColor: '#ef4444', bgColor: 'rgba(239, 68, 68, 0.1)' },
    info: { borderColor: '#2563eb', bgColor: 'rgba(37, 99, 235, 0.1)' }
  };
  
  const style = typeStyles[type] || typeStyles.note;
  
  return (
    <div style={{
      borderLeft: `4px solid ${style.borderColor}`,
      backgroundColor: style.bgColor,
      padding: '1rem 1.5rem',
      borderRadius: '0 8px 8px 0',
      margin: '1.5rem 0'
    }}>
      {children}
    </div>
  );
}

// Uso no MDX
<Callout type="warning">
  Esta é uma mensagem de aviso importante!
</Callout>
